# AGENTS.md

This project is a compiler server. It's made of multiple sub-projects.

## Compiler client

At `./client/rsh`.

An R package written in C. Communicates with the compiler server via gRPC. Uses LLVM ORC to load compiled code at runtime.

See `./client/rsh/README.md`.

## Compile server

At `./server`.

Written in Java 26. Uses Maven.

### Conventions

**IMPORTANT: Write idiomatic *new* Java. This includes:**

- Modules and module imports
- Records instead of POJOs
- `switch` expressions, `case Type e` and `instanceof Type e` (or better, `case Type(var a, ...)` and `instanceof Type(var a, ...)`)
- `var`
- `_`
- Triple-quoted multiline strings
- Markdown Javadoc
- Assume non-null by default. Explicitly annotate with `@Nullable` (from `org.jspecify.annotations`) when necessary.
- Annotate immutable collections with `@Unmodifiable` (from `org.jetbrains.annotations`), and readonly views of collections with `@UnmodifiableView` (from `org.jetbrains.annotations`)
- Use maintained dependencies, not deprecated or superseded ones (though dependencies should also be reliable).

The code is Google style, use `spotless` to reformat. Use the Jetbrains MCP server to lint.

#### Project-specific conventions

- The compiler must be non-deterministic. In particular, prefer `LinkedHashSet`/`LinkedHashMap` over `HashSet`/`HashMap`, only using the non-linked variants if they are never iterated on.
- If the user asks to write tests and the implementation is buggy, fix the implementation and test that the bug is fixed.

### Structure

- `./server/src/main/java/org/prlprg/`
    - `bc/` - R to bytecode compiler (main class `BCCompiler`)
    - `bc2c/` - Bytecode to C compiler (main class `BC2CCompiler`)
    - `bc2fir/` - Bytecode to FIŇė compiler (main class `BC2ModuleCompiler`)
    - `fir/` - Functional IR system with type checking and optimizations (see `./server/doc/FIR.md`)
    - `fir2c/` - FIŇė to C compiler (main class `FIR2CCompiler`)
    - `gen2c/` - Structures shared by both `bc2c` and `fir2c` (main class `CompiledModule`)
    - `parseprint/` - Parsing and printing utilities
    - `primitive/` - Miscellaneous R primitives (e.g., `Logical` represents R's "logical" which is a 3-way boolean. `Names` has utilities for reading and writing valid R symbols)
    - `rds/` - R data serialization format handling
    - `server/` - gRPC server and service implementation
    - `session/` - R session management and package handling
    - `sexp/` - R S-expression representation and manipulation
    - `util/` - Common utilities and helper classes

### Compilation Pipeline

```
R Function (SEXP) ‚Üí Bytecode (BC) ‚Üí C Code ‚Üí Native Object Code
                     ‚Üď              ‚Üď
                  Cache Layer   Cache Layer
```

### Testing Strategy

- **Snapshot integration tests**: Each test runs on example source code from `src/test/resources/org/prlprg/examples/`. It takes as input either the source itself and/or outputs of tests for earlier segments of the pipeline (e.g. the FIŇė to C test takes as input a FIŇė module encoding the example, which is produced by the FIŇė parser test for FIŇė examples and BC to FIŇė test for R examples). It produces output, which is either the next stage or end result of the pipeline, and saves it to a subdirectory of `src/test/snapshots/org/prlprg/snapshots/`.
- **Unit tests**: JUnit 5
- **Property tests**: JQwik
